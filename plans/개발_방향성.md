# 멀티 AI Agent 시스템 — 전체 개발 방향성

> 작성일: 2026-07-12  
> 언어: Node.js (JavaScript)  
> 자동화 엔진: Playwright  
> 지원 OS: macOS / Windows

---

## 1. 프로젝트 개요

### 목적
Gemini (https://gemini.google.com/app) 웹 UI를 Playwright로 자동화하여,
사용자가 정의한 **다단계 프롬프트 파이프라인**을 순차·반복 실행하고
각 단계의 결과를 다음 단계로 전달하는 **멀티 AI Agent 오케스트레이터**를 구축한다.

### 핵심 가치
- 코드 수정 없이 UI에서 프롬프트·워크플로우를 완전히 관리
- 단계 간 결과물 자동 연계 (이전 단계 출력 → 다음 단계 입력)
- 조건부 반복(루프) 지원 — 원하는 품질 기준을 충족할 때까지 특정 단계를 재실행
- Mac / Windows 동일 코드베이스로 실행 가능

## 2. 기술 스택

| 계층 | 기술 | 선택 이유 |
|------|------|-----------|
| 런타임 | **Node.js 20 LTS** | 크로스플랫폼, Playwright 공식 지원, references 코드베이스와 일치 |
| 브라우저 자동화 | **Playwright (chromium)** | Mac/Windows 동일 API, 퍼시스턴트 프로파일 지원, Shadow DOM 접근 |
| UI 서버 | **Express.js** | 경량, 빠른 REST API 구성 |

| 설정 저장 | **JSON 파일** (로컬) | 별도 DB 불필요, 이식성 우수 |
| 세션 관리 | **Playwright 퍼시스턴트 컨텍스트** | references의 gemini_session.js 패턴 재활용 |

```json
  "createdAt": "2026-07-12T00:00:00Z",
  "updatedAt": "2026-07-12T00:00:00Z"

```

#### 스텝 (Step)
```json
{
  "id": "step_001",
  "name": "키워드 분석",
  "prompt": "다음 주제에 대해 SEO 핵심 키워드 5개를 추출해줘: {{input}}",
  "variables": {
    "input": ""
  },
  "loop": {
    "enabled": true,
    "condition": "keyword",
    "conditionValue": "키워드",
    "maxRetries": 3,
    "retryDelayMs": 2000
  },
  "timeout": 120000,
  "newChat": true
}
```

#### 루프 조건 타입

| 조건 타입 | 설명 | 예시 |
|-----------|------|------|
| `none` | 반복 없음 (1회 실행) | — |
| `keyword` | 응답에 특정 키워드 포함 여부 | "성공", "완료" |
| `minLength` | 응답 최소 글자 수 | 500자 이상 |
| `regex` | 정규식 패턴 매칭 | `/\d{3}-\d{4}/` |
| `noError` | 에러 코드 없으면 통과 | — |

## 6. Gemini Agent 자동화 상세

### 6-1. references 코드 활용 전략

| 참고 파일 | 활용 내용 |
|-----------|-----------|
| `gemini_common.js` | DOM 셀렉터 (`SELECTORS`), SPA 준비 대기 (`waitForGeminiSpaReady`), 생성 완료 감지 (`isGeminiStillGenerating`) |
| `gemini_session.js` | 쿠키 내보내기/가져오기, 세션 스냅샷 관리 |
| `gemini_compose.js` | 프롬프트 입력 로직 (`tryFillGeminiComposer`), 응답 대기 (`waitForGeminiResponse`) |
| `workflow_control.js` | 워크플로우 단계 상태 관리 패턴, Abort 플래그 처리 |
| `bridge_protocol.js` | 로그 마커 포맷 (`[WORKFLOW]`, `[ERROR]`) — SSE 스트리밍에 동일 패턴 적용 |
| `google_login.js` | Google 로그인 셀렉터 목록 |
| `playwright_gemini_ask.js` | 단일 질문-응답 흐름의 기본 패턴 |
| `fallback_selectors.json` | DOM 셀렉터 폴백 목록 |

### 6-2. 새 채팅 시작 로직

```
1. gemini.google.com/app 접속 (퍼시스턴트 프로파일로 자동 로그인)
2. 좌측 상단 "새 채팅" 버튼 클릭 (또는 /app URL 재접속으로 초기화)
3. SPA 준비 대기 (waitForGeminiSpaReady)
4. 프롬프트 텍스트 입력 (tryFillGeminiComposer 패턴)
5. 전송 버튼 클릭
6. 응답 생성 완료 대기 (isGeminiStillGenerating → false)
7. 응답 텍스트 추출 및 반환
```

### 6-3. 에러 처리 & 재시도

- **셀렉터 폴백 체인**: 메인 셀렉터 실패 시 `fallback_selectors.json` 순차 시도
- **생성 타임아웃**: 기본 180초 (스텝별 설정 가능)
- **페이지 스톨 감지**: 응답이 오래 멈추면 자동 재시도 (최대 3회)
- **Abort 플래그**: 사용자가 UI에서 중단 시 플래그 파일로 즉시 종료

### 6-4. Gemini 응답 추출 방법

응답 텍스트 추출은 아래 우선순위로 시도한다:

```
1순위: 복사 버튼 클릭 → 클립보드 텍스트 읽기
        (가장 안정적, gemini_compose.js waitForGeminiResponse 패턴)

2순위: DOM 직접 파싱
        [data-test-id="model-response"] → innerText
        .model-response-text           → innerText

3순위: page.evaluate() 로 Shadow DOM 내부 탐색
```

## 7. 단계 간 데이터 흐름 (Result Pipe)

```
Step 1 실행 → 결과: "키워드: AI, 머신러닝, 딥러닝"
                    ↓
        context["step_001_output"] = "키워드: AI, 머신러닝, 딥러닝"
        context["prev_output"]     = "키워드: AI, 머신러닝, 딥러닝"
                    ↓
Step 2 프롬프트: "다음 키워드로 블로그 개요를 작성해줘: {{prev_output}}"
  → 변수 치환 후: "다음 키워드로 블로그 개요를 작성해줘: 키워드: AI, 머신러닝, 딥러닝"
                    ↓
Step 2 실행 → 결과: "1장: AI 개요 2장: 머신러닝..."
                    ↓
        context["step_002_output"] = "1장: AI 개요 2장: 머신러닝..."
        context["prev_output"]     = "1장: AI 개요 2장: 머신러닝..."
```

### 지원 내장 변수

| 변수 | 설명 |
|------|------|
| `{{prev_output}}` | 직전 단계의 출력 |
| `{{step_NNN_output}}` | 특정 단계(ID)의 출력 |
| `{{workflow_input}}` | 워크플로우 실행 시 최초 입력값 |
| `{{today}}` | 실행 날짜 (YYYY-MM-DD) |
| `{{step_index}}` | 현재 스텝 번호 (1부터 시작) |

## 8. 루프(반복) 실행 흐름

```
Step N 실행
     ↓
루프 조건 평가
     ├── 조건 충족 (또는 루프 없음) ──→ 다음 Step으로 진행
     └── 조건 미충족 & 재시도 가능  ──→ 딜레이 후 Step N 재실행
                                              ↑
                                    (최대 재시도 횟수 초과 시 에러 처리)
```

### 루프 활용 예시

| 시나리오 | 루프 조건 설정 |
|---------|--------------|
| 글자 수 500자 이상이 될 때까지 본문 작성 | `minLength: 500` |
| 응답에 "완료" 키워드가 나올 때까지 요약 | `keyword: "완료"` |
| JSON 형식 응답이 검증될 때까지 재시도 | `regex: /^\s*\{[\s\S]*\}\s*$/` |
| 에러 없이 성공할 때까지 재시도 | `noError` |

## 9. 세션 관리 전략

### 최초 실행 (로그인 필요)
1. Playwright 퍼시스턴트 프로파일로 브라우저 실행
2. `profiles/gemini-profile/` 디렉토리에 세션 자동 저장
3. 다음 실행 시 저장된 프로파일로 자동 로그인 (재로그인 불필요)

### 세션 만료 감지 & 복구
1. Gemini 로그인 상태 확인 (`geminiSignedInSelectors` 체크)
2. 미로그인 감지 시 UI에 알림 → 사용자가 브라우저에서 수동 로그인
3. 로그인 후 자동 재시작 옵션

### 쿠키 스냅샷 (선택 기능)
- `gemini_session.js`의 `exportCookies`/`importCookies` 패턴 활용
- `data/sessions/gemini-cookies.json`에 저장
- 여러 PC 간 세션 이식 가능

## 11. API 설계 (REST + SSE)

### 워크플로우 CRUD
```
GET    /api/workflows           ← 전체 워크플로우 목록
POST   /api/workflows           ← 새 워크플로우 생성
GET    /api/workflows/:id       ← 특정 워크플로우 조회
PUT    /api/workflows/:id       ← 워크플로우 수정
DELETE /api/workflows/:id       ← 워크플로우 삭제
```

### 스텝 CRUD
```
GET    /api/steps               ← 전체 스텝 목록
POST   /api/steps               ← 새 스텝 생성
GET    /api/steps/:id           ← 특정 스텝 조회
PUT    /api/steps/:id           ← 스텝 수정
DELETE /api/steps/:id           ← 스텝 삭제
```

### 실행 제어
```
POST   /api/run                 ← 워크플로우 실행 시작 (workflowId, input 전달)
POST   /api/abort               ← 실행 중단 요청
GET    /api/status              ← 현재 실행 상태 조회
GET    /api/events              ← SSE 스트림 (실시간 로그 & 상태)
```

## 13. 보안 고려사항

- Google 계정 비밀번호는 코드/파일에 절대 저장하지 않음
- 퍼시스턴트 프로파일 방식으로 자격증명 없이 세션 재사용
- `data/sessions/` 디렉토리 → `.gitignore` 추가 필수
- `profiles/` 디렉토리 → `.gitignore` 추가 필수
- API 서버는 `localhost`만 바인딩 (외부 노출 차단)
