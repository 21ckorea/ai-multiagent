# AI Multiagent

AI Multiagent는 사용자가 브라우저나 외부 도구에 직접 로그인하거나 개입하지 않아도, 구글 시트에 있는 콘텐츠 기획안을 바탕으로 **블로그 글을 자동으로 작성하고 썸네일/이미지를 생성하여 네이버 블로그에 발행**해 주는 통합 자동화 솔루션입니다. 

기획(Google Sheet) -> 작성(Gemini) -> 이미지 생성(Gemini Image) -> 발행(Naver Blog)의 전체 워크플로우를 완벽하게 자동화하여 사용자에게 최고의 편리함을 제공합니다.

---

## 🎯 주요 기능

1. **시트에서 주제 추출**: 구글 시트에 연동된 주제 및 기획안 데이터를 자동으로 읽어옵니다.
2. **AI 블로그 글 작성**: 읽어온 기획안을 바탕으로 구글 Gemini가 고품질의 블로그 초안(서론, 본문, 결론)을 자동 작성합니다.
3. **블로그 썸네일 및 본문 이미지 생성**: 글 내용에 맞는 썸네일과 본문 삽입용 이미지 여러 장을 Gemini Image를 통해 자동 생성합니다. 
4. **네이버 자동 포스팅**: 완성된 텍스트와 이미지들을 네이버 블로그 스마트에디터의 정확한 위치에 자동으로 찰떡같이 삽입하고 포스팅을 발행(비공개/공개)합니다.
5. **실시간 진행률 및 시트 완료 처리**: 전체 진행 과정을 웹 UI로 제공하며, 포스팅이 성공적으로 완료되면 구글 시트의 상태를 '완료'로 자동 업데이트합니다.

## 🛠️ 설치 방법 (Installation)

### 1. 필수 조건
- **Node.js** (v18 이상 권장)
- **Chrome / Playwright**: 브라우저 자동화를 위한 Playwright 의존성이 필요합니다.

### 2. 패키지 설치
```bash
# 저장소 복제
git clone https://github.com/21ckorea/ai-multiagent.git
cd ai-multiagent

# 의존성 패키지 설치
npm install

# Playwright 브라우저 설치 (설치되지 않은 경우)
npx playwright install
```

### 3. 환경 변수 및 인증 정보 설정
본 프로그램은 Google API 및 네이버 로그인을 위해 인증 파일이 필요합니다.  
`credentials` 폴더 내에 인증 정보를 구성하세요. (GitHub에는 보안상 업로드되지 않습니다.)

- **Google 서비스 계정 키**: 구글 시트 접근용 `cred_*.json` 파일
- 기타 구글 Gemini API를 사용하기 위한 설정 파일

> **참고**: `credentials`, `references`, `profiles`, `data` 폴더 및 `.env` 파일은 깃허브에 업로드되지 않으므로 개별 환경에 맞게 직접 구성해야 합니다.

## 🚀 사용 방법 (Usage)

### 1. 서버 실행
프로젝트 루트 디렉토리에서 다음 명령어를 실행하여 서버를 시작합니다.
```bash
npm start
```
> 포트포워딩 또는 다른 설정으로 실행 중인 프로세스를 다시 시작하려면 `npm run restart` 명령어를 사용할 수 있습니다.

### 2. 웹 UI 접속
브라우저를 열고 `http://localhost:3000` (기본 포트) 에 접속합니다.
UI에서 **전체 실행** 버튼을 클릭하면 위에서 설명한 1단계부터 5단계까지의 모든 과정이 전자동으로 진행됩니다. 특정 스텝부터 재시작하거나 단일 스텝만 실행하는 것도 가능합니다.

## ⚙️ 포트 및 기타 설정 변경 방법

기본적으로 웹 UI와 API 서버는 **3000번 포트**를 사용합니다. 
포트를 변경하거나 추가적인 환경 설정을 하고 싶다면 아래 방법을 따르세요.

### 1. `.env` 파일을 통한 변경
프로젝트 최상단 루트 디렉토리에 `.env` 파일을 생성하고 아래와 같이 환경변수를 입력합니다. (파일이 없다면 새로 만들어주세요.)

```env
# 실행 포트 변경 (예: 8080으로 변경)
PORT=8080
```
설정을 변경한 후에는 반드시 서버를 재시작해야 변경 사항이 반영됩니다.

### 2. 명령어 직접 지정
서버를 실행할 때 포트를 직접 지정할 수도 있습니다.
```bash
# Mac / Linux
PORT=8080 npm start

# Windows (cmd)
set PORT=8080 && npm start
```
